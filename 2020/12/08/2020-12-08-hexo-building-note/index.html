使用 Hexo 架設部落格

前言

此部落格是用 Hexo 來架設，並佈署在 Github 上。
主題我是使用了 icarus 再自己寫 CSS 客製化 ，這個主題有中文文件，內容也相當完整。

藉此文章將架設流程及常用指令記錄起來，以便快速查閱。
想知道更多關於 Hexo 的介紹與教學，我在本文最後提供了相關資源供參考。

Hexo 的特色

  • 使用 Markdown 語法撰寫。
  • 由台灣人開發，中國協作者多，中文資源多。
  • 可以部屬在 Github 上，就不需要處理另外花錢買網域。

架設流程

準備環境

安裝 Node.js 和 Git。 (Node.js 建議安裝 LTS 穩定版)

前期安裝

  1. 安裝 hexo CLI

  2. 建立 hexo 雛形

  3. 安裝 npm 套件管理工具

Hexo 常用指令

查詢 hexo 指令

建立新文章

開啟模擬伺服器

建立(編譯)靜態網站

部署到遠端伺服器

清除暫存檔案

佈署到 Github

若要放在 Github 的根目錄
每個帳戶只有一個能放在最根目錄的專案

儲存庫名稱： {用戶名稱}.github.io
網址： https://{用戶名稱}.github.io

流程

  1. 連接遠端 Github 資料庫
  2. 修改 _config.yml
  3. 安裝 hexo 自動部署工具
  4. 佈署

關於 post

我在 post 會用到的 yaml

我自己會加進 scaffolds/post.md 裡面，這樣使用 $ hexo new post 時就會自動生成

※　在文章的 yaml 沒有設定 date 時，預設的文章日期是以檔案生成時間為準。

非單一 categories 和 tags 的寫法

手動設定文章的省略段落

在想要顯示的段落為止加上<!-- more -->

post 的檔案命名

  • 最好可以自訂 年-月-日-文章標題 的格式來寫： 2020-10-15-這是文章標題.md
    (檔名會影響該文章的網址，所以建議使用英文)
  • 檔案前加上下底線 _ 可以有隱藏效果，文章可以藉此當作草稿功能 _2020-10-15-這是草稿.md
